Aplicativo

Aqui você verá a documentção do sistema do aplicativo da IS SAÚDE Gestão Hospitalar, que está disponível para download nas plataformas iOS e Android.

Quem somos?

Fundado em 2009, a ISGH, criadora do software IS SAÚDE, nasce junto ao inicio da implantação do padrão TISS. Padrão este, criado pela agencia nacional de Saúde com o intuito de padronizar codificação e descrição de procedimentos médicos, codificação de material e medicamento hospitalar. Com a frequente dificuldade em encontrar empresas de software com a agilidade necessária para se adaptar ao novo padrão, surgiu a necessidade instintiva em solucionar problemas e entregar faturamento de empresas. Por haver sido desenvolvido por pessoas com grande capacidade e experiência em gerenciamento hospitalar, o Sistema IS SAÚDE segue sempre o caminho mais curto para alcançar o objetivo. Nossos acertos foram ficando cada vez mais pontuais, e a AGILIDADE, passou a ser rotina em nosso aplicativo.

Descrição da documentação

O que é oferecido no Aplicativo

  1. Login (acesso com CPF e Senha)
    • link de Esqueceu a Senha ( Caso o usuário esquece a senha com este link ele consegue gerar uma nova senha de acesso).
    • link de Nova Conta ( Com este link o usuário consegue acessar a tela de cadastro de novo usuário).
    • Botão Entrar nesse botão possui verificações de segurança, verifica se os dados estão vazios, verifica CPF Válido, se o CPF e a Senha confere. Validações nescessárias para a segurança do aplicativo e do usuário.
  2. Cadastro de Usuário
    • O clinte irá inserir as informações ( Nome Completo, Email, CPF, Data Nascimento, Telefone, Sexo e escolher um Convênio deselado ).
    • O cadastro possui validações e campos formatodas como CPF, DATA, e TELEFONE.
    • O Servidor retorna as mensagems de validação, Erros e Sucessos dependendo do StatusCode.
  3. Recuperação de Senhas
    • No recuperar senha, temos um campo para adicionar o CPF, O Servidor recebe o cpf, valida e retorna uma mensagem dizendo que a senha foi enviada para o email.
    • No caso de erro por conta de alguma validação retorna uma mensagem do servidor.

Métodos e Recursos

Todas as requisições devem conter os seguintes HEADERS.

{
    "Content-Type": "application/json"
    "cnpj": "CNPJ CLIENTE"
    "senha": "Senha de Segurança de Autenticação"
}
